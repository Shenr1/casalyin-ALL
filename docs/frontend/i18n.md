# 国际化规范（i18n）

> 来源：frontend_i18n.md
> 最后更新：2026-04-11

---

## 技术方案

- 库：`react-i18next` + `i18next` + `i18next-browser-languagedetector`
- 配置文件：`src/i18n.ts`
- 语言存储：`localStorage` key `app_language`
- fallback：`zh-CN`
- 支持语言：`zh-CN`、`en`、`es-ES`

## 翻译文件

| 语言 | 文件 |
|------|------|
| 简体中文 | `src/locales/zh-CN.json` |
| 英文 | `src/locales/en.json` |
| 西班牙文 | `src/locales/es-ES.json` |

---

## 核心规范

### 禁止写死任何用户可见文案

所有用户可见文字必须走 `t()`，包括：message.success/error/warning、Modal 标题/内容/按钮、Table 列标题、Form label、placeholder、空状态提示、页面标题等。

```tsx
// ❌ 错误
message.success('创建成功')
<Button>提交</Button>
placeholder="请输入名称"

// ✅ 正确
const { t } = useTranslation();
message.success(t('agronomist.createSuccess'))
<Button>{t('common.submit')}</Button>
placeholder={t('agronomist.namePlaceholder')}
```

### 非组件代码中使用 i18n 实例

```ts
import i18n from '../i18n';
i18n.t('request.notLoggedInTitle')
```

### key 命名规范

- 通用文案放 `common.*`，页面专属文案放 `页面名.*`
- 示例：`common.save`、`product.createSuccess`、`store.loadFailed`

### 三语言同步更新

每次新增 key，必须在 `zh-CN.json`、`en.json`、`es-ES.json` 三个文件同步添加。

### lint 检查

`npm run lint` 会执行 `scripts/check-chinese.mjs` 扫描硬编码中文，存在则失败。
