## 第一期：认证与会话管理

> TC-AUTH-001（注册流程）已移除：系统无独立注册页，第一次邮箱登录即自动注册，注册即登录。

### TC-AUTH-002 登录与 Token 机制

**步骤：**
1. 访问 `/login`，输入 B 角色邮箱 `123456@qq.com`，点「Send Code」
2. 填入验证码 `123456`，点「Sign In」
3. 验证跳转到 `/dashboard`
4. 点右上角头像展开下拉，点「Log out」退出
5. 浏览器回退，尝试访问 `/products`

**验证点：**
- [ ] 登录后 token 存在（Cookie 或 LocalStorage）
- [ ] 退出后 token 清除
- [ ] 退出后回退访问受保护页面 → 跳转 `/login`，不展示任何数据
- [ ] 错误密码登录 → 提示错误，不跳转

**异常分支：**
- 直接访问 `/dashboard`（未登录） → 跳转 `/login`
- Token 手动篡改后请求 API → 返回 401，前端跳转登录页

### 草稿状态枚举

| draftStatus | 枚举 | 前端 Tag | 允许操作 |
|-------------|------|---------|---------|
| 0 | DRAFT | 草稿（橙色）| 用户可编辑、可保存、可提交 |
